# Test Documentation — Ledger Module (`ledger.c`)

This document describes the verification suite used to test the binary data persistence module (`ledger.c`). The component extracts transaction history blocks via opaque indexers and dumps the data into structured binary records on disk.

## Implemented Test Cases

### 1. File Database Allocation and Trashing
* **Target Function:** `ldg_save_ledger`
* **Objective:** Validate that providing a clean system path successfully initializes and truncates the database file.
* **Verification:** Invokes the save routine with an empty transaction engine history handle, confirming a return code of `ERR_NONE` and verifying the existence of a zero-byte file on disk.

### 2. OS Directory Protection Trapping
* **Target Function:** `ldg_save_ledger`
* **Objective:** Verify that OS file access errors or invalid directory hierarchies are caught safely.
* **Verification:** Supplies an unreachable directory path (`"/invalid/path/ledger.bin"`) and asserts that the function catches the failure and returns the structural error token `ERR_ORD`.

### 3. Handle Pointer Null Guard
* **Target Function:** `ldg_save_ledger`
* **Objective:** Ensure that calls containing unallocated matching engine context pointers are blocked before initiating disk I/O operations.
* **Verification:** Sends a parameter value mapped as `mtc_handle = NULL` and verifies an immediate error return of `ERR_ORD`.

### 4. Binary Record Parity Deserialization
* **Target Function:** `ldg_save_ledger`
* **Objective:** Validate byte-accurate data serialization of an executed transaction (`mtc_transaction_t`) generated through the production pipeline.
* **Verification:** Populates a temporary CSV file with an explicit execution pattern, runs the parser and matching engine to generate a trade record, and invokes `ldg_save_ledger`. Reopens the output file via native `fopen/fread` stream routines, reads the raw byte struct, and asserts field-by-field parity (`trade_id == 1`, `price == 50.0`, `quantity == 5`).

### 5. Batch Stress Sequential Serialization
* **Target Function:** `ldg_save_ledger`
* **Objective:** Ensure I/O streaming performance and data stability when committing a large volume of data rows in a single batch file write.
* **Verification:** Generates a dataset containing exactly 100 valid transactions inside the matching repository, triggers a bulk write via `ldg_save_ledger`, and asserts that the operation completes with a successful `ERR_NONE` status.

### 6. Byte Stream Insertion Order Preservation
* **Target Function:** `ldg_save_ledger`
* **Objective:** Verify that binary structure padding rules preserve chronological record positioning during sequential reads.
* **Verification:** populates five incremental transaction records inside the matching engine cache. Commits the database layer to disk, reads back the file blocks into an array of structs using the official shared type from `matching.h`, and asserts that the entry loaded at slot index `[i]` matches the data pattern generated by the engine.
